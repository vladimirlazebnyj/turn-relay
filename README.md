# turn-relay

Прокидываем WireGuard или V2Ray трафик через TURN серверы ВК-звонков. Работает так: пакеты шифруются по DTLS 1.2, потом параллельными потоками уходят на TURN сервер через TCP или UDP по протоколу STUN ChannelData. Оттуда по UDP летят на твой VPS, расшифровываются и попадают в WireGuard. Логин и пароль для TURN генерируются автоматически из ссылки на звонок.


## Что понадобится

Что нужно:

1. **Ссылка на ВК звонок** — создай свой (нужен аккаунт ВК) или загугли `"https://vk.com/call/join/"`. Ссылка живёт вечно, пока сам не нажмёшь "завершить звонок для всех".
2. **Или ссылка на Яндекс Телемост** — `"https://telemost.yandex.ru/j/"`. Лучше не гугли — при поиске видно, что ты подключился к чужой конференции.
3. **VPS с установленным WireGuard.**
4. **Для Android** — скачай Termux из F-Droid.

## Настройка сервера

<details><summary>Нужно использовать tmux — без него процесс умрёт при закрытии SSH</summary>

Запусти tmux на сервере:

```bash
tmux new -s relay
```

Внутри сессии запусти команду сервера (ниже). Потом нажми `Ctrl+B`, затем `D` — tmux свернётся, процесс продолжит работать. Вернуться обратно:

```bash
tmux a -t relay
```

</details>

Скачиваем бинарник, в примере используется `server-linux-amd64`:

```bash
curl -L -o server https://github.com/cacggghp/turn-relay/releases/latest/download/server-linux-amd64 && chmod +x server
```

Запускаем:

```bash
./server -listen 0.0.0.0:56000 -connect 127.0.0.1:<порт wg>
```

### Установка как демон (systemd)

Создай файл `/etc/systemd/system/turn-relay.service`:

```
[Unit]
Description=Turn Relay Service
After=network.target

[Service]
Type=simple
ExecStart=/opt/turn-relay/server-linux-amd64 -listen 0.0.0.0:56000 -connect 127.0.0.1:<wg_port>
KillMode=process
Restart=always
RestartSec=5
User=nobody
Group=nogroup
StandardOutput=append:/var/log/turn-relay/turn-relay.log
StandardError=append:/var/log/turn-relay/turn-relay_error.log
SyslogIdentifier=turn-relay

[Install]
WantedBy=multi-user.target
```

Где `/opt/turn-relay/server-linux-amd64` — путь к бинарнику, `<wg_port>` — порт WireGuard.

```
systemctl daemon-reload
systemctl enable turn-relay.service
systemctl start turn-relay.service
```

### Docker

Образ публикуется в GitHub Container Registry:

```
docker pull ghcr.io/cacggghp/turn-relay:latest
docker tag ghcr.io/cacggghp/turn-relay:latest relay
```

На Linux-сервере, где xray или WireGuard слушает локально, удобнее через host network:

```
docker run --rm --network host -e CONNECT_ADDR=127.0.0.1:<порт wg> relay
```

Если нужен bridge mode:

```
docker run --rm -p 56000:56000/udp -e CONNECT_ADDR=<ip хоста>:<порт wg> relay
```

Собрать образ вручную:

```
git clone https://github.com/cacggghp/turn-relay.git
cd turn-relay
docker build -t turn-relay .
```

Переменная окружения **CONNECT_ADDR** — адрес WireGuard (обязательная), например `192.168.1.10:51820`.

Пример запуска:

```
docker run -p 56000:56000/udp -e CONNECT_ADDR=192.168.1.10:51820 turn-relay
```

## Настройка клиента

### Android

Готовые приложения — [смотри выше](#android).

**Через Termux (альтернативный способ):**

- В конфиге WireGuard меняем адрес сервера на `127.0.0.1:9000`, MTU ставим 1280.
- **Добавляем Termux в исключения WireGuard. Жмём "Сохранить".**

В Termux:

```
termux-wake-lock
```

Это не даёт телефону уходить в глубокий сон — на ночь ставь на зарядку. Чтобы снять блокировку:

```
termux-wake-unlock
```

Скачиваем бинарник (архитектура `arm64` — самая популярная):

```bash
curl -L -o client https://github.com/cacggghp/turn-relay/releases/latest/download/client-android-arm64 && chmod +x client
```

Запускаем:

```
./client -listen 127.0.0.1:9000 -peer <ip сервера wg>:56000 -vk-link <VK ссылка>
```

Или через Яндекс:

```
./client -udp -turn 5.255.211.241 -peer <ip сервера wg>:56000 -yandex-link <Ya ссылка> -listen 127.0.0.1:9000
```

**Если после включения VPN в терминале вылезают ошибки DNS — в настройках WireGuard включи VPN только для нужных приложений.**

## Если что-то не работает

Флагом `-turn` можно вручную указать адрес TURN сервера. Это должен быть сервер ВК, Макса или Одноклассников (для ВК ссылки) или Яндекса (для Яндекс ссылки).

Если не работает TCP — попробуй добавить флаг `-udp`.

Для более стабильного подключения в один поток — добавь `-n 1` (ограничение скорости ~5 МБит/с для ВК).

Для ручного прохождения капчи — флаг `-manual-captcha`.

По умолчанию капча проходится так: сначала автоматическая попытка, потом автопопытка через пазл-слайдер, и только потом ручной режим.

## Яндекс Телемост

**UPD. ТЕЛЕМОСТ ЗАКРЫЛИ**

У Яндекса нет ограничений скорости в отличие от ВК, поэтому по умолчанию стоит `-n 1`. Если увеличить это число — конференция переполняется фейковыми участниками и тебя могут временно забанить по IP.

В режиме `-udp` скорость обычно выше.

Большинство диапазонов IP TURN серверов Яндекса не работают — указывай вручную через `-turn`.

## VLESS-режим

Можно использовать VLESS через флаг `-vless`. В этом режиме вместо UDP-пакетов пробрасываются TCP-соединения через TURN-туннель с помощью KCP и smux.

### Настройка
1. На VPS установить Xray с VLESS inbound
2. Запустить `server` с флагом `-vless`
3. На клиенте запустить `client` с флагом `-vless`
4. Настроить Xray/v2rayN клиент с VLESS outbound на `127.0.0.1:9000`

### Сервер (VPS)
```
./server -listen 0.0.0.0:56000 -connect 127.0.0.1:443 -vless
```

#### Docker
```
docker run -p 56000:56000/udp -e CONNECT_ADDR=127.0.0.1:443 -e VLESS_MODE=true turn-relay
```

### Клиент
```
./client -peer <ip сервера>:56000 -vk-link <VK ссылка> -listen 127.0.0.1:9000 -vless
```

## Direct mode

С флагом `-no-dtls` можно отправлять пакеты без DTLS-обфускации и подключаться к обычным серверам WireGuard. Осторожно — это может привести к бану от ВК или Яндекса.
